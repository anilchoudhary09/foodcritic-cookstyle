1. Adding a Service Exception (@restricted_services)

Rule Enforced: BARC017 - NoSystemServices

When to Use

When a cookbook needs to manage a service that would otherwise be blocked.

Format

@restricted_services = {
  'service_name' => ['cookbook1', 'cookbook2'],  # Specific cookbooks allowed
  'other_service' => [],                          # ALL cookbooks allowed
}

Adding a New Exception

To allow a specific cookbook to manage a service:

'new_service' => ['my_cookbook_name'],  # CHG1234567890 - Your change number

To allow ALL cookbooks to manage a service:

'new_service' => [],  # CHG1234567890 - Your change number

Example

@restricted_services = {
  # ... existing entries ...

  # NEW: Allow app_my_team_service cookbook to manage custom-app service
  'custom-app' => ['app_my_team_service'],  # CHG1234567890 Application service

  # NEW: Allow ANY cookbook to manage my-generic-service
  'my-generic-service' => [],  # CHG1234567891 Generic service for all
}
Important Notes:
  • Always include the Change Request number as a comment
  • Service names are case-sensitive
  • Partial matches work (e.g., 'tomcat@' matches tomcat@instance1)

2. Adding an /etc Path Exception (@etc_whitelist)

Rule Enforced: BARC005 - EtcBlacklist

When to Use

When a cookbook needs to modify a file under /etc/ that would otherwise be blocked.

Format

@etc_whitelist = {
  '/etc/path/to/file' => ['cookbook1', 'cookbook2'],  # Specific cookbooks
  '/etc/some/directory/' => [],                        # ALL cookbooks (note trailing /)
}

Adding a New Exception

To allow a specific cookbook to modify an /etc path:

'/etc/my/config.conf' => ['my_cookbook'],  # CHG1234567890 - Description

To allow ALL cookbooks to modify an /etc path:

'/etc/common/config.d/' => [],  # CHG1234567890 - Description

Example

@etc_whitelist = {
  # ... existing entries ...

  # NEW: Allow my_app_cookbook to modify application config
  '/etc/myapp/config.yaml' => ['my_app_cookbook'],  # CHG1234567890 App config

  # NEW: Allow my_app_cookbook to modify any file in /etc/myapp/conf.d/
  '/etc/myapp/conf.d/' => ['my_app_cookbook'],  # CHG1234567890 App config directory

  # NEW: Allow ANY cookbook to use /etc/custom/
  '/etc/custom/' => [],  # CHG1234567891 Custom configs for all
}
Important Notes:
  • Paths ending with / match any file in that directory
  • Exact file paths require exact matches
  • Always check @etc_blacklist first - if path is blacklisted, it takes precedence unless whitelisted
  • Include Change Request number as comment

3. Blocking an /etc Path for ALL Cookbooks (@etc_blacklist)

Rule Enforced: BARC005 - EtcBlacklist

When to Use

When an /etc path should NEVER be modified by ANY cookbook (security/system critical files).

Format

@etc_blacklist = [
  '/etc/critical/path',
  '/etc/critical/directory/',
]

Adding a New Blacklist Entry

@etc_blacklist = [
  # ... existing entries ...
  '/etc/new/critical/file',  # Reason for blacklisting
]
Important Notes:
  • Blacklist takes precedence unless specifically whitelisted for a cookbook
  • Use sparingly - this blocks ALL cookbooks
  • Cannot be overridden by @etc_whitelist unless cookbook is explicitly listed

4. Adding a System Service (@system_services)

Rule Enforced: BARC017 - NoSystemServices

When to Use

When a new system service should be protected from ALL cookbooks.

Format

@system_services = %w[
  service1
  service2
  sshd
  ntpd
]

Adding a Protected System Service

@system_services = %w[
  # ... existing entries ...
  new_protected_service  # Reason for protection
]
Important Notes:
  • System services are blocked for ALL cookbooks
  • To allow a cookbook to manage a system service, add to @restricted_services with the cookbook name
  • Examples: sshd, ntpd, sssd, sendmail

5. Adding an Attribute Exception (@restricted_attributes)

When to Use

When a cookbook needs to set a restricted attribute.

Format

@restricted_attributes = {
  'attribute_name' => ['cookbook1', 'cookbook2'],
}

Example

@restricted_attributes = {
  # ... existing entries ...

  'enable_custom_feature' => ['my_cookbook'],  # CHG1234567890 Feature flag
}

6. Adding a Command Exception (@cmd_whitelist)

When to Use

When a cookbook needs to run a specific shell command that would otherwise be flagged.

Format

@cmd_whitelist = {
  'exact command string' => ['cookbook1', 'cookbook2'],
  'another command' => [],  # ALL cookbooks allowed
}

Example

@cmd_whitelist = {
  # ... existing entries ...

  # Allow specific cookbook to run yum install
  'yum install -y mypackage' => ['my_cookbook'],  # CHG1234567890

  # Allow all cookbooks to run this command
  'systemctl daemon-reload' => [],  # CHG1234567891
}
Important Notes:
  • Commands must match exactly
  • Multi-line commands are supported (use heredoc-style)

7. Adding a Platform Cookbook (@platform_cookbook_whitelist)

When to Use

When a cookbook is a platform/infrastructure cookbook that needs to bypass certain checks.

Format

@platform_cookbook_whitelist = [
  'platform_cookbook_1',
  'platform_cookbook_2',
]

Example

@platform_cookbook_whitelist = [
  # ... existing entries ...

  'my_platform_cookbook',  # CHG1234567890 New platform cookbook
]

Change Request Process

Required Information

Field Description Example
Change Request Number ServiceNow CR number CHG1234567890
Cookbook Name Exact name from metadata.rb app_my_team_myapp
Exception Type Service, /etc path, command, etc. Service Exception
Justification Why is this exception needed Application requires custom service
Security Review Required for sensitive paths/services Approved by Security Team

Format for Comments

# Standard format
'resource' => ['cookbook'],  # CHG1234567890 Brief description
For Middleware Services - Approval Required:
Email: PaaSMiddlewareEngineering@barclayscorp.com
Attach email approval to Change Request. All unapproved cookbooks will be removed without notice.

Testing Your Exception

After adding an exception, test it:

# Navigate to your cookbook
cd cookbooks/your_cookbook

# Run Cookstyle with the rules
cookstyle . --config ../b-cookstyle-rules/.rubocop.yml

# Expected: No BARC violations for the whitelisted resource
Success Criteria: No BARC violations for the resources you whitelisted.

Common Scenarios

Scenario Section to Modify Code Example
New Application Service @restricted_services 'my-app-service' => ['app_my_team_myapp'], # CHG1234567890
Application Config File @etc_whitelist '/etc/myapp/config.yaml' => ['app_my_team_myapp'], # CHG1234567890
Directory of Configs @etc_whitelist '/etc/myapp/conf.d/' => ['app_my_team_myapp'], # CHG1234567890
Multiple Cookbooks Same Resource @etc_whitelist or @restricted_services '/etc/shared/config' => ['cookbook1', 'cookbook2', 'cookbook3'], # CHG1234567890
Any Cookbook Can Use Any Hash section 'generic-service' => [], # CHG1234567890 All cookbooks allowed

File Structure Summary

# rules.rb Structure

@system_services = %w[           # Array - Protected services (blocked for all)
  sshd
  ntpd
]

@restricted_attributes = {       # Hash - Attribute restrictions
  'attr_name' => ['cookbook'],
}

@restricted_services = {         # Hash - Service exceptions
  'service' => ['cookbook'],     # Specific cookbook allowed
  'service2' => [],              # ALL cookbooks allowed
}

@etc_whitelist = {               # Hash - /etc path exceptions
  '/etc/path' => ['cookbook'],   # Specific cookbook allowed
  '/etc/dir/' => [],             # ALL cookbooks allowed (note /)
}

@etc_blacklist = [               # Array - /etc paths blocked for ALL
  '/etc/passwd',
  '/etc/shadow',
]

@cmd_whitelist = {               # Hash - Command exceptions
  'command' => ['cookbook'],
}

@platform_cookbook_whitelist = [ # Array - Platform cookbooks
  'platform_cookbook',
]
