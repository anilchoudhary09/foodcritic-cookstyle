4. How BARC Rule Exceptions Work

4.1 Exception Flow in barc_cops.rb

4.2 Key Methods in BarcRulesEngine

Method Used By Description
etc_allowed?(path, cookbook) BARC005, BARC005a Checks if /etc path is whitelisted for cookbook
service_allowed?(service, cookbook) BARC017 Checks if service is allowed (empty array = all allowed)
service_needs_review?(service, cookbook) BARC018 Checks if service needs review flagging
platform_bypass?(cookbook) ALL rules Bypasses all rules if in @platform_cookbook_whitelist

4.3 Adding New Exceptions

To add a new exception:
  1. Open cookbooks/b-cookstyle-rules/rules.rb
  2. Find the appropriate variable (lines 1-1767 only)
  3. Add your exception with a change ticket comment
  4. Commit and push - exception applies immediately
# Example: Allow /etc/myapp.conf for my-cookbook (BARC005)
@etc_whitelist = {
  '/etc/myapp.conf' => ['my-cookbook'],  # CHG0012345
}

# Example: Allow custom service for specific cookbooks (BARC017/018)
@restricted_services = {
  'my-custom-service' => ['cookbook1', 'cookbook2'],  # CHG0012345
}

# Example: Allow service for ALL cookbooks
@restricted_services = {
  'my-service' => [],  # CHG0012345 - empty = all cookbooks
}
