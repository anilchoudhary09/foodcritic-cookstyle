2. Code Optimization Opportunities

2.1 Extract Common Command Check Pattern

Current Issue: Multiple cops repeat the same command-checking pattern with different command lists.

The following cops share nearly identical on_block patterns:

Cop Pattern Optimization
BARC007, BARC008, BARC010, BARC012, BARC019, BARC020 Check execute/bash for forbidden commands Extract to shared module

Proposed Optimization:

# Add to CookbookHelper module
module CommandChecker
  def check_forbidden_commands(node, unix_cmds, win_cmds = [], opts = {})
    return unless execute_resource?(node)
    cmd = extract_command(node)
    return unless cmd

    cmd_lower = cmd.downcase
    return unless unix_cmds.any? { |c| cmd =~ /\b#{c}\b/i } ||
                  win_cmds.any? { |c| cmd_lower.include?(c) }
    add_offense(node.send_node)
  end

  def execute_resource?(node)
    %i[execute bash script powershell_script batch].include?(node.send_node.method_name)
  end
end

# Simplified cop:
class Barc007NoSelinux < RuboCop::Cop::Base
  include CookbookHelper
  include CommandChecker

  MSG = 'BARC007: Do not manipulate SELinux configuration.'
  FORBIDDEN_CMDS = %w[chcon semanage setenforce setsebool togglesebool setfiles].freeze

  def on_block(node)
    return if BarcRulesEngine.platform_bypass?(cookbook_name) || BarcRulesEngine.selinux_bypass?(cookbook_name)
    check_forbidden_commands(node, FORBIDDEN_CMDS)
  end
end

Estimated Reduction: ~100-150 lines

5. Why to Keep BARC Rules

✅ Key Reasons to Keep BARC Rules:

5.1 Enterprise Bypass/Exception Mechanism

The core value of BARC rules is the whitelist/bypass mechanism driven by rules.rb:

Feature BARC Rules Built-in Cookstyle
Per-cookbook exceptions ✅ @etc_whitelist, @restricted_services, etc. ❌ Global only
Per-path exceptions ✅ @etc_whitelist['/etc/myapp/'] => ['cookbook1'] ❌ Not supported
Per-service exceptions ✅ @restricted_services['mysql'] => [] ❌ Not supported
Change ticket tracking ✅ Comments in rules.rb ❌ Not built-in
Platform cookbook bypass ✅ @platform_cookbook_whitelist ❌ Manual exclusion per cop

6. Recommendations

6.1 Code Optimizations (Recommended)

DO:
  1. Extract CommandChecker module for common command detection
  2. Move repeated helper methods to CookbookHelper:
    • extract_service_name()
    • extract_package_name_attr()
    • extract_path_attribute()
  3. Consider memoization for cookbook_name calls

6.2 Replacements (Optional)

CONSIDER (only if bypass not needed):
  1. BARC026 → Chef/Correctness/CookbookUsesNodeSave
DO NOT REPLACE:
  1. BARC022 - broader detection than built-in
  2. BARC004 - checks commands + paths, not just keys in code
  3. BARC024 - has regex validation + cookbook name in URL check
  4. BARC011 - covers Unix + Windows, built-in is PowerShell only
  5. BARC023 - requires existence, not just validation

6.3 Final Recommendation

Summary:
  • Keep all 36 BARC rules - the enterprise bypass/exception mechanism is the core value
  • Apply code optimizations - can reduce ~165-215 lines (11-15%)
  • Optionally enable built-in cops as additional checks - they can complement BARC rules
  • BARC026 is the only rule that could be fully replaced by built-in
